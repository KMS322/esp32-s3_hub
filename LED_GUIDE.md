# 허브 WS2812 LED 동작 가이드 (gatt_server_usb)

고객 지원·현장 진단용. **정상 동작은 고정 또는 단순 깜빡임**, **이상 시에는 빨강 SOS + 구분색 펄스**로 구분한다. (레거시 `gatt_server_251105/led_color.txt`와 같은 원리, 이 펌웨어는 **초기 설정이 USB CDC**이므로 BLE 광고 단계는 **USB 대기**로 매핑했다.)

- **하드웨어**: GPIO 48, WS2812 1개, 기본 밝기 `HUB_LED_DEFAULT_BRIGHTNESS`(40/255) — `hub_led_init()`에서 설정.
- **구현**: `main/hub_led.c`, `main/hub_led.h` — FreeRTOS 태스크 한 개가 패턴을 담당.

---

## 1. 정상 동작 (상태별)

| 상황 | LED |
|------|-----|
| NVS에 Wi-Fi 없음 — 아직 PC로 설정 안 함 | **빨강 고정** |
| USB CDC 열기 전 / 초기화 전 — USB로 `wifi:...` 받기 대기 | **청록**, 약 **1초 주기** 깜빡임 (0.5s ON / 0.5s OFF) |
| USB CDC 준비됨 — `usb connected\n` 보낸 뒤 PC에서 문자열 입력 대기 | **청록 고정** |
| Wi-Fi 연결 시도 중 (NVS 또는 USB/BLE로 받은 계정) | **주황** 깜빡임 (~0.4s ON/OFF) |
| 서버 등록 확인 `POST /check/hub` 진행 중 | **노랑** 깜빡임 |
| MQTT 브로커 연결 시도 중 | **초록** 깜빡임 (느리게, 고정 초록과 구분) |
| MQTT 연결 완료 후 명령 수신 대기 등 정상 유휴 | **초록 고정** |
| `connect:devices` 후 BLE 스캔·연결 시도 | **파랑** 깜빡임 |
| BLE 스캔·연결 후 MQTT로 목록 보고 성공 직후 | **초록 고정**으로 복귀 |

**참고 버전과의 대응**

- 예전: “GATT 광고 모드 → 청록 깜빡임” → **이번 펌웨어: USB 설정 대기 → 청록 깜빡임/청록 고정**
- 예전: “데이터 수신 중 파랑 고정” → **이번 펌웨어: 스캔 중 파랑 깜빡임, 유휴는 초록 고정** (원격 명령 대기는 초록으로 통일)

## 2. 오류 패턴 (빨강 → 구분색)

한 사이클: **빨강 짧은 펄스 3회** → 짧은 간격 → **아래 표의 색 펄스 N회** → 긴 간격 → 반복.  
정상 상태로 복구되면 `hub_led_set_mode()` 호출 시 오류 표시가 해제된다.

| 종류 | 구분색 (펄스 횟수) | 의미 |
|------|-------------------|------|
| HTTP 실패 | 노랑 (3) | `/check/hub` 실패, 응답 없음, 또는 `mqtt server ready` 없음 |
| MQTT 초기화 실패 | 보라 (3) | `mqtt_init()` 실패 |
| MQTT 연결 실패 | 보라 (2) | 브로커 연결 시간 초과, 스캔 결과 MQTT 미연결 등 |
| MQTT 전송 실패 | 보라 (1) | ready/state/scan 등 MQTT publish 실패 |
| MQTT 전송 오류 | 보라 (3) | `MQTT_EVENT_ERROR`(TCP 등) |
| Wi-Fi 실패 | 주황 (3) | NVS 재시도 초과 또는 USB 경로에서 연결 재시도 초과 |
| BLE 스캔 실패 | 파랑 (3) | 스캔에서 디바이스 없음 |
| BLE 연결 실패 | 파랑 (2) | 스캔은 됐으나 연결 실패, BLE disconnect 실패 |
| BLE 전송 실패 | 파랑 (1) | `MODE:*` 전송 실패 |
| NVS 저장 실패 | 청록 (3) | `save_nvs` 실패 |
| USB CDC 실패 | 마젠타 (3) | USB 초기화 실패 (레거시 문서에 없음, USB 전용) |
| 메모리 할당 실패 | 빨강 (5)만 연속 | MAC 버퍼 등 `malloc` 실패, JSON 생성 실패 시 |

## 3. 고객 안내 문구 예시

- **빨강만 켜짐**: 공장 초기 상태이거나 Wi-Fi가 비어 있음. USB로 PC 연결 후 `wifi:SSID,비밀번호,이메일` 형식으로 보내세요.
- **청록이 천천히 깜빡임**: USB 준비 대기 중. 케이블·드라이버 확인.
- **청록 고정**: PC에서 Wi-Fi 문자열을 보내기만 하면 됨.
- **주황 깜빡임**: 라우터·비밀번호 확인.
- **노랑 깜빡임**: 인터넷·서버 연결 확인.
- **초록 깜빡임**: 클라우드(MQTT) 연결 중.
- **초록 고정**: 정상. 앱/서버에서 명령 가능.
- **파랑 깜빡임**: 주변기기(Tailing_HUB) 찾는 중.
- **빨강 + 다른 색 번갈 패턴**: 위 표의 오류 유형 — 해당 색으로 원인 구분.

---

## 4. 소스에서의 위치

- 모드/오류 API: `hub_led.h`
- 패턴 렌더링: `hub_led.c`
- 상태머신 연동: `main.c` 각 `STATE_*`
- MQTT 이벤트 오류: `mqtt.c` (`MQTT_EVENT_ERROR`)

---

## 5. 변경 이력 요약

- **MQTT 발행 성공 시** 청록/초록 깜빡임 제거: 유휴 **초록 고정**과 혼동 방지.
- 모든 `ws2812_*` 직접 호출을 `hub_led_*`로 통합(스캔 등).
